🚀 내일의 계획 (Day 2: Personalization & Memory)

내일은 사용자의 '준비 시간'을 단순히 고정값(30분)으로 두지 않고, 실제 데이터를 기록하여 학습하는 **'개인화 엔진'**의 기초를 다집니다.

1. 사용자 데이터 저장 (Persistence)

매일 아침 실제로 일어난 시간과 집에서 나간 시간을 기록하는 로컬 파일(JSON) 시스템 구축.

preparation_logs.json을 통해 요일별, 상황별 준비 시간 데이터 누적.

2. 준비 시간 학습 로직

단순 평균값이 아닌, 최근 3일간의 데이터에 가중치를 두는 이동 평균(Moving Average) 알고리즘 적용.

"월요일은 평소보다 준비가 5분 더 걸리시네요"와 같은 인사이트 도출.

3. 예외 상황 처리 고도화

날씨 API(OpenWeatherMap 등) 연동 검토: 비/눈 올 때 소요 시간 가중치(+20%) 자동 적용.
